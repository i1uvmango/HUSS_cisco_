# Bridge-X Q&A - 발표 준비용

## 프로젝트 의의 및 가치

---

### Q1. 왜 분쟁지역 청소년을 대상으로 했나요?

**A:** 분쟁지역 청소년은 전쟁 트라우마, 가족 상실, 강제 이주 등으로 **심각한 심리적 고통**을 겪지만, 전문 상담 인력과 자원이 절대적으로 부족합니다.

성인들 보다 심리적 상처에 의한 회복탄력성이 더 낮습니다

 UNHCR 보고에 따르면 난민 중 정신건강 서비스를 받는 비율은 **5% 미만**입니다. Bridge-X는 이 격차를 **AI 기술로 메우고**, 언제 어디서든 접근 가능한 심리 지원을 제공하기 위해 개발되었습니다.

---

### Q2. AI 상담이 실제 전문 상담을 대체할 수 있나요?

**A:** AI는 전문 상담을 **대체하는 것이 아니라 보완**합니다. Bridge-X의 AI는:
- 24시간 즉시 접근 가능한 **1차 심리 지원**
- 감정 표현을 유도하고 경청하는 **공감적 대화**
- 위기 상황 시 **전문 상담사에게 즉시 연결**

AI는 상담의 문턱을 낮추고, 진정 도움이 필요한 순간에 전문가 개입을 빠르게 연결하는 **다리 역할**을 합니다.

---

### Q3. 왜 Cisco Webex를 사용했나요?

**A:** Cisco Webex는:
- **기업/의료급 보안**: 엔터프라이즈 수준의 암호화 및 개인정보 보호
- **글로벌 인프라**: 어떤 지역에서든 안정적인 화상 통화 가능
- **Bot API 지원**: 자동화된 미팅 생성 및 알림 시스템 구축 가능
- **신뢰성**: 전 세계 수백만 사용자가 검증한 플랫폼

분쟁지역의 불안정한 네트워크 환경에서도 **신뢰할 수 있는 연결**을 제공합니다.

---

### Q4. 위기 감지는 어떻게 작동하나요? 오탐(False Positive)은 없나요?

**A:** Google Gemini AI가 대화 중 자해/자살 관련 키워드와 문맥을 분석합니다.
- **키워드 기반**: "죽고 싶다", "I want to die" 등 직접적 표현
- **문맥 분석**: 극심한 절망감, 희망 없음 표현

오탐 가능성은 있지만, **위기 상황에서는 과잉 대응이 과소 대응보다 안전**합니다. 미팅이 생성되어도 상담사가 상황을 판단하여 대응합니다.

---

### Q5. 대화 내용은 저장되나요? 프라이버시는 어떻게 보호하나요?

**A:** 
- **대화 원문은 저장하지 않습니다** - 세션 종료 시 즉시 삭제
- **비식별화된 감정 데이터만 저장** - 감정 태그, 주요 주제 등 구조화된 정보
- **상담 개선 목적으로만 활용** - 개인 식별 불가능

청소년이 안심하고 이야기할 수 있는 **신뢰 환경**을 최우선으로 설계했습니다.

---

### Q6. 기존 정신건강 앱과 차별점은 무엇인가요?

**A:** 

| 기존 앱 | Bridge-X |
|---------|----------|
| 일반 대중 대상 | **분쟁지역 청소년 특화** |
| 자가 진단 위주 | **AI 대화 + 전문가 연결** |
| 수동 예약 필요 | **위기 시 자동 연결** |
| 채팅만 제공 | **Webex 화상 상담 통합** |

Bridge-X는 **AI의 즉시성**과 **인간 상담의 전문성**을 결합한 하이브리드 모델입니다.

---

### Q7. 실제 현장에서 사용 가능한가요? 확장 계획은?

**A:** 현재는 프로토타입 단계이지만, 다음 단계로:
1. **다국어 확장**: 아랍어, 우크라이나어, 터키어 등 난민 주요 언어 지원
2. **오프라인 모드**: 불안정한 네트워크 환경 대응
3. **NGO 파트너십**: UNHCR, IRC 등 현장 단체와 협력
4. **상담사 대시보드**: 다수 케이스 관리 기능

궁극적으로 **글로벌 난민 심리 지원 플랫폼**으로 확장하고자 합니다.

---

### Q8. 왜 Google Gemini AI를 선택했나요?

**A:** 
- **다국어 능력**: 한국어, 영어, 아랍어 등 자연스러운 처리
- **감정 이해도**: 문맥과 뉘앙스를 파악하는 고급 언어 모델
- **안전성**: 유해 콘텐츠 필터링 내장
- **비용 효율성**: 대규모 서비스에도 합리적인 API 비용

---

### Q9. 이 프로젝트의 사회적 임팩트는 무엇인가요?

**A:** 
1. **접근성 혁신**: 어디서든 심리 지원 접근 가능
2. **골든타임 확보**: 위기 순간 즉각 개입으로 생명 보호
3. **스티그마 감소**: 앱 기반 상담으로 정신건강 서비스 진입 장벽 낮춤
4. **데이터 기반 개선**: 익명화된 데이터로 난민 심리건강 연구 기여

---

### Q10. 프로젝트 이름 "Bridge-X"의 의미는 무엇인가요?

**A:** 
- **Bridge (다리)**: 단절된 청소년과 심리적 안정, 미래를 **연결**
- **X (미지수/확장)**: 청소년 각자의 **무한한 가능성**과 Cisco Webe**X**의 기술력

> "단절을 넘어, 가능성으로 이어지는 연결" - Bridge-X의 핵심 가치입니다.

## 기술적 Q&A

---

### T1. 시스템 아키텍처는 어떻게 구성되어 있나요?

**A:** 
```
Frontend (Next.js 15) → Backend (NestJS) → AI (Gemini) + Webex API
                              ↓
                      PostgreSQL Database
```

- **Frontend**: Next.js 15, React 19, TailwindCSS
- **Backend**: NestJS (TypeScript), TypeORM
- **Database**: PostgreSQL
- **AI**: Google Gemini 2.5 Flash API
- **Video**: Cisco Webex Meetings API

---

### T2. API는 어떤 구조인가요?

**A:** RESTful API 구조:

| Endpoint | Method | 기능 |
|----------|--------|------|
| `/api/chat` | POST | AI 대화 메시지 전송 |
| `/api/chat/greeting` | GET | 초기 인사말 + 세션 생성 |
| `/api/summary` | POST | 감정 분석 요약 생성 |
| `/api/counseling/request` | POST | Webex 상담 요청 |
| `/api/users` | POST | 사용자 등록 |

---

### T3. 위기 감지 로직은 어떻게 구현했나요?

**A:** AI 프롬프트에 위기 감지 규칙을 내장:

```typescript
// AI가 위기 상황 감지 시 응답 시작에 태그 삽입
if (response.includes('[RISK_DETECTED]')) {
    // 자동으로 Webex 미팅 생성
    await webexService.createCounselingMeeting(userId, summaryId, true);
}
```

- **키워드 감지**: "죽고 싶다", "I want to die" 등
- **문맥 분석**: Gemini AI가 자연어로 위험도 판단
- **즉시 대응**: 감지 즉시 상담방 자동 생성

---

### T4. Webex Bot은 어떻게 동작하나요?

**A:**
1. **트리거**: 위기 감지 또는 유저 요청 시
2. **API 호출**: Webex Meetings API로 미팅 생성
3. **알림**: 미팅 생성 시 상담사에게 자동 푸시 알림
4. **연결**: 미팅 URL을 사용자에게 제공

```typescript
const meeting = await this.client.post('/meetings', {
    title: isUrgent ? '[긴급] 심리 상담' : '[상담 요청] 심리 상담',
    start: now,
    end: oneHourLater,
    enabledJoinBeforeHost: true
});
```

---

### T5. 세션 관리는 어떻게 하나요?

**A:** 
- **In-Memory Store**: 대화는 서버 메모리에만 저장 (Map 구조)
- **UUID 기반 세션**: 각 대화에 고유 세션 ID 부여
- **자동 삭제**: 감정 분석 요약 생성 후 즉시 세션 삭제
- **비식별화 저장**: 감정 태그, 주제만 DB에 저장

```typescript
// 세션 종료 시
this.chatService.clearSession(sessionId);
```

---

### T6. 데이터베이스 스키마는?

**A:** 주요 테이블:

| 테이블 | 설명 |
|--------|------|
| `users` | 익명 사용자 (닉네임만 저장) |
| `ai_summaries` | 감정 분석 결과 (태그, 점수, 리스크) |
| `counseling_sessions` | 상담 세션 (Webex 미팅 ID, 상태) |

**대화 원문은 저장하지 않음** → 프라이버시 보호

---

### T7. 보안은 어떻게 적용했나요?

**A:**
- **HTTPS 통신**: 모든 API 통신 암호화
- **CORS 설정**: 허용된 도메인만 접근
- **환경 변수**: API 키는 `.env`로 분리, Git에 포함 안함
- **Webex 보안**: 엔터프라이즈급 E2E 암호화
- **세션 휘발성**: 대화 원문 미저장

---

### T8. 프론트엔드 상태 관리는?

**A:** 
- **React useState**: 로컬 컴포넌트 상태
- **세션 ID**: 채팅 시작 시 백엔드에서 발급받아 유지
- **실시간 업데이트**: 메시지 전송 후 응답 즉시 렌더링

별도 상태 관리 라이브러리(Redux 등) 없이 **간결한 구조** 유지

---

### T9. 확장성은 어떻게 고려했나요?

**A:**
- **모듈화 아키텍처**: NestJS 모듈 단위로 기능 분리
- **서비스 분리**: AI, Webex, Chat 각각 독립 서비스
- **환경 변수**: 설정만 변경하면 다른 AI/화상 서비스로 교체 가능
- **다국어 지원**: Gemini AI가 입력 언어 자동 감지 후 응답
